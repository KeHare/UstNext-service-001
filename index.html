既存の就活サイトは、4年制大学生を中心に設計されています。

現場の先生と一人人事の善意で回ってきました。

学生の声
企業の本音が見えない。マイナビは大学生向けで、自分の学科に合う求人が探しづらい。

専門学校生は、専門性・働き方・自分軸の3軸で職を選ぶ。検索型のサイトでは、学校で学んだ強みを言語化できず、自分の本来の選択肢に出会えない。

企業の声
専門学校に届く選択肢が少なく、求人票を出しても、想いが伝わりづらい。

専門学校とつながる手段が全くないわけではない。それでも、選択肢が限られていて、求人票という書類だけでは社風や育てる文化までは伝えきれない。学校との関係構築には時間と人手がかかる。

200人の学生の動きを、紙とExcelで追い切れない。学生のチャットの中身まで見届けられない。

就職課の現場は、推薦状の作成、面談記録、企業対応、保護者対応で手一杯。学生がどの企業とどう話しているか、トラブルが起きる前に気づく仕組みが無い。

UstNext で、何が変わるか

専門学校生・専門学校にとって、具体的に何が変わるのか。

機能の話ではなく、現場で起きる変化をそのまま並べます。読み終わった瞬間に「これは必要だ」と感じていただけるよう、忖度なく書きました。

専門学校生にとって、変わる5つのこと

  1. 自分の学科に合う求人が、最初から並ぶ。 検索の言葉を考えなくても、IT学科・医療事務学科・建築学科など学科タブで自分向けの求人が一覧に。大学生向けサイトで埋もれがちだった専門卒求人が、最初から見つかります。
  2. テキストが苦手でも、自分を見せられる。 自己PRが書けなくても、GitHubコード・YouTube動画・スキルシートで実力を伝えられる。専門学校で作ってきたものが、そのまま履歴書になります。
  3. 失敗を学校に知られたくない、を解決する匿名スカウトモード。 学科の小さなコミュニティで「あの会社で落ちた」が広まる怖さを、匿名応募で解消。マッチした後だけ実名が開示される設計だから、安心してチャレンジできます。
  4. 企業の本音（社風・育てる文化）が、求人票より深く見える。 「何をやって／なぜやって／どうやっているのか」の3点フォーマット、社員の動画、SNS連携。文化の解像度が上がるから、入ってから「思っていたのと違う」が減ります。
  5. 学校が見守ってくれている、という安心感。 先生が学生×企業のチャットを見守れる仕組み。トラブルが起きる前に気付ける。「一人で就活している感覚」が消えます。

専門学校（就職課）にとって、変わる5つのこと

  1. 200人の動きが、1画面で俯瞰できる。 学科別の応募・内定・未活動の状況がダッシュボードで一覧。紙とExcelで月次集計していた業務が、毎朝30秒のチェックに置き換わります。
  2. 学生×企業のチャットを見守れる。 不適切なやり取り、トラブルの兆しを早期発見。書き込み権限はなく、閲覧のみ。「監視」ではなく「保護」の機能設計で、学生・保護者の信頼を損ないません。
  3. 学校公認の枠組みで動ける。 求人を学校が確認・推薦できる、学校から学生への発信もできる。学校としてのお墨付き運用が、はじめてオンライン上で完結します。
  4. 既存の校内システムと併存できる。 J-Navi⁺ など既存の学校管理システムを置き換えるのではなく、企業発信のリッチさと学生UIを補完する形で併用できます。導入リスクが小さく、既存資産が活きます。
  5. 学生・保護者に「ここで就活していい」と伝えられる。 学校が選んだプラットフォームを学生に勧められる。保護者にも「学校が見守れる仕組み」と説明できる。学校としての信頼が可視化されます。
UstNext の答え

専門学校生の就活を、3 者が同じテーブルで進める。

企業は想いを発信できる。学生は自分軸で選べる。学校はその動きを見守れる。
3つが連動して初めて、専門学校生が「自分に合う」と納得できるマッチングが成立します。

学生視点

専門学校生が、自分の言葉で動ける。

スマホ・モバイル世代の指の動きに合わせて、就活初心者でも触れる設計。
  • FEATURE 01
    スワイプで求人を探す
    TikTok・Tinder世代に親和。左右スワイプで「気になる／スキップ」を直感的に。検索の言葉が思いつかない学生でも、まずカードを見るところから始められます。
  • FEATURE 02
    匿名スカウトモード
    学校名・氏名を伏せ、スキルだけで企業に見てもらえる。マッチ後に実名開示。「失敗が学校に知られたくない」専門学校の小さなコミュニティでも、安心してチャレンジできます。
  • FEATURE 03
    GitHub・自己PR動画 (YouTube/Loom) リンク
    テキストでの自己PRが苦手でも、作品・コード・話す姿で実力を伝えられる。IT・クリエイティブ・接客系まで、専門学校生の多様な学びを表現できます。
  • FEATURE 04
    モバイル下部タブ・ナビゲーション
    ホーム／スワイプ／求人／気になる／チャットを下部固定タブで提示。LINE・Instagramの感覚で迷わず使える。スマホ片手で就活が完結します。
企
企業視点

中小企業が、想いと現場を学生に届ける。

採用ノウハウが豊富でない一人人事でも、企業の魅力を伝えるシナリオが用意されています。
  • FEATURE 01
    「何をやっているのか／なぜやるのか／どうやっているのか」企業ページ
    求人票では伝わらない「会社の想い」「事業の意義」「働き方」を、書きやすい3点フォーマットで発信。中小企業でも、ミッションを明文化することで学生の文化マッチを生み出します。
  • FEATURE 02
    URL取得・ハロワ求人票OCR（順次実装）
    既存の求人ページURLを貼るだけで情報を取得。ハローワーク求人票のPDF・画像からOCRで自動入力する機能も順次実装中。一人人事の手入力工数を最小化します。
  • FEATURE 03
    若者雇用促進法・青少年雇用情報に標準対応
    研修制度、メンター、平均勤続年数、産休育休取得率まで、法令で求められる若者雇用情報をフォーマット化。法令違反リスクを下げながら、誠実な企業として学生に届きます。
  • FEATURE 04
    紹介動画・SNS連携・ブランドビジュアル
    YouTube動画、Twitter / Instagram / Facebook / TikTok / LinkedIn のSNS連携、カバー画像とロゴ。中小企業でも、現場の空気をそのまま学生に届けるための受け皿があります。
校
就職課視点

就職課が、学生の動きを見守れる。

紙とExcelでは追い切れなかった200人の動きを、1画面で俯瞰します。
  • FEATURE 01
    学科別・就活進捗ダッシュボード
    全学科の動きを1画面で俯瞰。学科ごとの応募状況、内定者数、未活動の学生まで一目で把握。就職課主任の朝のチェック動線にハマります。
  • FEATURE 02
    学生×企業のチャットを閲覧できる「見守りモード」
    学生と企業のやり取りを、就職課が閲覧のみ可能（書き込み権限なし）。トラブルの早期発見、不適切なやり取りの抑止、ハラスメントの予防に。「もし何かあれば、先生が気づける」という安心が学生にも届きます。
  • FEATURE 03
    推薦・承認ワークフロー
    学校としてお墨付きを与える求人、学生が個別に応募する求人を整理。学校経由の推薦と、学生の自発的な動きの両方に対応します。
  • FEATURE 04
    学生・求人の一元管理
    学生一覧、求人管理、チャット監視を就職課のサイドメニューに統合。情報をあちこち探さない、1つのプラットフォームで完結する設計。

なぜ、専門学校生に必要か

専門学校生は、「マッチング」ではなく「納得」を求めている。

大学生のように選択肢の中から「効率良く選ぶ」のではなく、専門学校生は「自分の専門性を活かせる場所で、長く働ける確信」を得たい。納得のいく就職には、3者の連動が不可欠です。

専門性
学んだスキルが活きる現場で働きたい

学科別求人フィルタ、スキルシート、GitHub・動画PR、企業ページの「どうやっているのか」で、技術と現場の解像度を上げる。

働き方
自分のリズムで、長く働ける場所で働きたい

勤務時間・育成体制・社風・リモート可否など、求人票の数値だけでは見えない働き方の解像度を、企業ページとチャットで深く知れる仕組み。

納得
「ここで働く意味」を自分の言葉で言えるまで決めたくない

企業の想い、学校の見守り、学生の自分軸が同じテーブルに乗ることで、「親や先生に納得してもらえる」言葉ができる。

導入の流れ

3 ステップで、専門学校・企業・学生が動き始めます。

1

就職課ヒアリング

就職課の現状業務、学生の規模、企業との接点をヒアリング。学校独自の運用に合わせて、見守り設計とアカウント発行を行います。

2

企業ページ作成

提携企業の企業ページを共同で作成。既存求人ページのURL取得、ハロワ求人票OCR（順次実装）で、入力工数を最小化します。

3

学生招待・運用開始

学校からのアナウンスで学生がアカウント登録。企業ページとスワイプから就活が始まります。就職課は学生の動きを見守りながら、必要に応じて介入します。

よくあるご質問

導入前に多い、3つの質問。

既存の学校管理システム（J-Navi⁺など）との併存はできますか？

可能です。UstNextは「学生・企業・学校が同じテーブルで動く」プラットフォームであり、既存の校内管理システムを置き換えるものではありません。学生のリッチな発信と、企業の想いの可視化、就職課の見守りを担う形で併用できます。

学生のチャット見守り機能はプライバシー上問題ありませんか？

就職課は閲覧のみ可能で、書き込み権限はありません。画面上に「見守りモード」が明示され、学生・保護者にも事前に同意いただく運用を推奨しています。トラブルの早期発見と学生保護を目的とした、教育機関としての責任を果たすための設計です。

料金体系を教えてください。

学校向けは規模別の年間契約をご用意しています。ベータ版期間中の特別プランもございますので、個別にご相談ください。

専門学校生だけが対象ですか？高校生や大学生は？

UstNextは専門学校生に特化したプラットフォームです。専門性 × 働き方 × 自分軸という専門学校生独自の選び方を、最初から設計の中心に置いています。高校生・大学生向けの就活サービスとは別物として位置付けています。

順次実装中の機能はいつ使えるようになりますか？

ハロワ求人票OCR、HP自動入力、推薦状PDF出力など、現場の声を反映した機能を順次リリース予定です。導入企業・学校には優先的にロードマップを共有し、ご要望を反映します。
